# Siri 语音控制钱包（iOS 快捷指令搭建指南）

通过 iPhone「快捷指令」App + 服务端 AI 语意解析，实现语音操作学习奖励钱包（步骤以 iOS 26 为准，iOS 17+ 界面基本一致）：

> 「嘿 Siri，打开零花钱」→ Siri：「已打开，要拿钱吗？」→ 你：「给她加 5 元，因为今天作业全对」→ Siri：「已加 5 元，备注"今天作业全对"，当前余额 23 元。」

## 前置条件

1. 服务端 `.env` 已配置 `SIRI_API_TOKEN`（生成方式：`openssl rand -hex 24`），并已重启容器。
2. `GET /api/health` 返回 `"siriConfigured": true`。
3. AI 模型已在管理后台配置（自然语言指令需要它；固定金额指令和查余额不需要）。

以下步骤中的占位符请替换为你自己的值：

- `https://your-domain.example/your-path` → 你的站点地址（不带末尾斜杠）
- `<SIRI_API_TOKEN>` → `.env` 里的真实 token

## API 一览

| 端点 | 方法 | 用途 |
|---|---|---|
| `/api/siri/wallet/command` | POST | 自然语言指令（服务端 AI 解析），体：`{"text": "..."}` |
| `/api/siri/wallet/adjust` | POST | 固定金额加/扣钱，体：`{"amountYuan": 5, "note": "可选"}`（负数为扣） |
| `/api/siri/wallet` | GET | 查余额 |

三个端点都需要请求头 `Authorization: Bearer <SIRI_API_TOKEN>`（`Bearer` 后一个空格），都返回 `speech` 字段——可直接朗读的中文整句。

自然语言指令支持：加钱（"加 5 元，奖励作业全对"）、扣钱（"扣 2 元，上课讲话"）、查余额（"还有多少钱"）。金额、动作白名单、单次上限（100 元）均在服务端强制校验；无关内容会被解析为"没听懂"，不会动钱包。

## 关于指令命名（重要，先读）

**不要把指令叫「打开钱包」**——「钱包」是 Apple 官方钱包 App 的中文名，对 Siri 说「打开钱包」大概率会直接打开 Apple 钱包 App 而不是运行快捷指令。建议用不冲突的名字，如「打开零花钱」「小金库」「奖励钱包」。指令名就是 Siri 唤醒语，随时可改（长按快捷指令 → 重新命名），改名不影响服务端。

## 指令一：「打开零花钱」（主指令，自然语言）

### 新建

1. 打开「快捷指令」App → 底部「快捷指令」标签 → 右上角 **+**。
2. 点顶部快捷指令名称（或名称旁的 ˅）→ **重新命名** → 输入 `打开零花钱`。

### 动作 1：要求输入

3. 点「添加动作」（或底部搜索框），搜索 **要求输入**，选中。
4. 点动作里的「提示」，输入：`已打开，要拿钱吗？`
   （Siri 会朗读这句话然后等你说话；想换台词直接改这里，无需改服务端。）
5. 点动作左侧 **>** 展开，确认「输入类型」为 **文本**（默认）。

### 动作 2：获取 URL 内容

6. 搜索添加 **获取 URL 内容**。
7. URL 填：`https://your-domain.example/your-path/api/siri/wallet/command`
8. 点动作左侧 **>** 展开：
   - 「方法」改为 **POST**。
   - 「头部」→ **添加新头部**：键 `Authorization`，值 `Bearer <SIRI_API_TOKEN>`。
   - 「请求体」保持 **JSON** → **添加新字段** → **文本**：
     - 键：`text`
     - 值：点值输入框，在键盘上方的变量栏选 **「提供的输入」**（即动作 1 的听写结果；没看到就点「选择变量」再点动作 1 的输出）。

### 动作 3：获取词典值

9. 搜索添加 **获取词典值**（自动接上一步）。
10. 「键」填：`speech`

### 动作 4：朗读文本

11. 搜索添加 **朗读文本**，确认朗读对象是上一步的 **词典值**（默认自动连接）。
12. 可展开打开「等待完成」。

### 首次运行授权（必做一次）

13. 点右下角 **▶** 手动运行：在提问输入框打字 `还有多少钱`，点完成。
14. 弹出「想要访问 your-domain.example」时选 **始终允许**（选「允许一次」会导致 Siri 每次追问，打断语音流程）。
15. 听到朗读余额即成功。之后即可全程语音使用。

## 指令二：「零花钱加五元」（可选，一句话直达、零延迟）

不追问、不走 AI，适合最常用的固定金额。新建快捷指令，命名 `零花钱加五元`：

1. **获取 URL 内容**
   - URL：`https://your-domain.example/your-path/api/siri/wallet/adjust`
   - 展开：方法 **POST**；头部 `Authorization` = `Bearer <SIRI_API_TOKEN>`
   - 请求体 JSON → 添加新字段 → **数字**：键 `amountYuan`，值 `5`
     （扣钱用负数如 `-2`；可再加文本字段 `note` 写固定备注）
2. **获取词典值**：键 `speech`
3. **朗读文本**

同样首次手动运行并选「始终允许」。

## 指令三：「查零花钱」

1. **获取 URL 内容**
   - URL：`https://your-domain.example/your-path/api/siri/wallet`
   - 展开：方法保持 **GET**；头部 `Authorization` = `Bearer <SIRI_API_TOKEN>`
2. **获取词典值**：键 `speech`
3. **朗读文本**

## 在其他设备上用

- **HomePod**：HomePod 不直接运行快捷指令，靠「个人请求」转发到你的 iPhone 执行。开启：家庭 App → 长按你的 HomePod → 设置（齿轮）→「Siri」下的 **个人请求** → 开启（需 iPhone 与 HomePod 同一网络、同一 Apple ID）。唤醒语与 iPhone 上完全一样。
- **AirPods / CarPlay**：即连即用，无需额外设置。
- **Apple Watch**：快捷指令详情 → ⓘ → 「在 Apple Watch 上显示」后可在表上语音触发。

## 故障排查

| 现象 | 原因与处理 |
|---|---|
| Siri 打开了 Apple 钱包 App | 指令名与系统词冲突，换名字（见命名节） |
| Siri 说「出现错误」/ 返回 401 | `Authorization` 头拼错：`Bearer` + 一个空格 + token，无换行、无多余空格 |
| 返回 503 | 服务端 `SIRI_API_TOKEN` 未配置或容器未重启 |
| 回复「AI 模型还没配置好」 | 管理后台的 AI 模型配置缺失；只影响自然语言指令 |
| 每次运行都弹「是否允许访问」 | 首次授权选了「允许一次」；重新运行一次选 **始终允许** |
| 回复「没听清金额」 | 听写没识别出数字，重说一遍带明确数字：「加五元」 |
| 无任何朗读 | 检查动作 3 的键是否为 `speech`、朗读动作是否连接了词典值 |

## 安全说明

- Token 等同支付口令：只保存在自己手机的快捷指令里，不要分享快捷指令给他人。
- 若怀疑泄露：修改云端 `.env` 中的 `SIRI_API_TOKEN` 并重启容器，旧 token 立即作废（记得同步更新快捷指令头部）。
- 所有语音操作以 `adjust` 类型入账，孩子端钱包流水和管理后台账本都能看到金额与备注，可随时核对；误操作用反向指令补回即可。
